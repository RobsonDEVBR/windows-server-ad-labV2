Ao promover O MBR1 a Controlador de domínio a parte de DNS ficou travada 
como solução usei o comando Install-ADDSDomainController -DomainName "robson.local" -InstallDns -Credential $cred
Este é o comando que garante que os dois servidores estão sincronizados. Dominar a saúde dessa replicação local é o pré-requisito exato para quando formos conectar essa sua infraestrutura com a nuvem (Azure AD/Entra ID) no seu projeto de arquitetura híbrida.

Abra o PowerShell como Administrador e digite:

PowerShell
repadmin /replsummary
O que procurar: Na tabela que aparecer, olhe as colunas fails e error. Se tudo estiver zerado (0), a comunicação está perfeita e o banco de dados foi copiado sem nenhuma corrupção.

🗺️ Teste 3: A Validação de Rotas (DNS)
Como você forçou a instalação do DNS no comando (-InstallDns), a rede precisa saber disso. Abra o CMD ou PowerShell e digite:

DOS
nslookup robson.local
O que procurar: A resposta agora deve listar dois IPs em vez de um só. Você deve ver o 10.10.10.10 (DC1) e também o IP fixo que você deu para o seu MBR1. Isso prova que se um servidor cair, as máquinas da rede saberão automaticamente que podem bater na porta do outro para fazer login.
Provas MBR 2.9 a 3.2
